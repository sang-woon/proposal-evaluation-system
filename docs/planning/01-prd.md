## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

현재 제안서 평가는 마치 **엑셀 시트를 여러 명이 각자 작성한 후, 한 사람이 일일이 손으로 더하는 것**과 같습니다. 평가위원들이 각자 점수를 입력하면, 담당자가 엑셀에서 수작업으로 집계해야 합니다. 이 과정에서:
- 집계 시간이 오래 걸림
- 계산 실수가 발생할 수 있음
- 평가서 양식을 맞추기 어려움

### 1.2 목표 상황 (To-Be)

평가위원이 웹에서 직접 점수를 입력하면, **자동으로 계산되고, 바로 출력할 수 있는 깔끔한 평가서가 완성**되는 상황. 마치 설문조사를 하면 자동으로 통계가 나오는 것처럼, 평가하면 바로 결과가 집계됩니다.

### 1.3 왜 지금인가? (Why Now)

- 수기 평가의 불편함이 누적됨
- 업무 효율화 필요성 증가
- 결과 집계 시 오류 방지 필요

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "외부 전문가 평가위원" |
| 나이/직업 | 40-60대 / 대학 교수, 연구원, 업계 전문가 |
| 핵심 니즈 | 빠르고 정확하게 평가하고 싶다 |
| 불편 포인트 | 복잡한 시스템 사용, 수기 작성의 번거로움 |
| 사용 상황 | 평가장에서 노트북으로 평가 진행 |

### 2.2 페르소나의 하루 (시나리오)

김 교수님은 오늘 정부 R&D 과제 평가를 위해 평가장에 도착했습니다. 예전에는 종이 평가서를 받아 손으로 점수를 쓰고, 나중에 담당자가 엑셀로 집계하느라 30분 넘게 기다렸습니다.

오늘은 다릅니다. 노트북을 열고 웹 주소를 입력하니 바로 평가 화면이 나옵니다. 제안서별로 평가 항목이 정리되어 있고, 각 항목에 점수만 입력하면 됩니다. 5개 제안서 평가를 마치니, "인쇄" 버튼을 누르자 A4 평가서가 깔끔하게 출력됩니다. 서명란에 서명만 하면 끝!

"이렇게 편할 수가!" 김 교수님은 집계 결과를 기다릴 필요 없이 바로 퇴근할 수 있게 되었습니다.

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (접속)
- [x] 사용자로서, **로그인 없이 바로 평가 화면에 접속**할 수 있어야 한다. 왜냐하면 빠른 평가 시작이 중요하기 때문이다.
- [x] 사용자로서, **평가할 제안서 목록을 확인**할 수 있어야 한다. 왜냐하면 어떤 제안서를 평가해야 하는지 알아야 하기 때문이다.

### FEAT-1: 평가 결과 자동 집계 + 평가서 출력 (MVP)
- [ ] 평가위원으로서, **평가항목별로 점수를 입력**할 수 있어야 한다. 왜냐하면 각 항목의 점수가 최종 평가의 근거가 되기 때문이다.
- [ ] 평가위원으로서, **입력한 점수의 합계를 자동으로 확인**할 수 있어야 한다. 왜냐하면 수작업 계산 없이 정확한 결과가 필요하기 때문이다.
- [ ] 평가위원으로서, **A4 크기로 평가서를 출력**할 수 있어야 한다. 왜냐하면 서명을 위한 종이 문서가 필요하기 때문이다.
- [ ] 관리자로서, **모든 평가위원의 점수를 한눈에 집계**할 수 있어야 한다. 왜냐하면 최종 결과 산출이 핵심 목적이기 때문이다.

### FEAT-2: 평가항목 관리 (v2 이후)
- [ ] 관리자로서, 평가항목을 추가/수정/삭제할 수 있어야 한다.

### FEAT-3: 제안서 관리 (v2 이후)
- [ ] 관리자로서, 제안서 정보를 등록하고 관리할 수 있어야 한다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-1: 평가 결과 자동 집계 + A4 평가서 출력
- FEAT-0: 바로 접속 가능한 웹 인터페이스

### 5.2 옵션 (시간 허용 시)
- 평가 의견 텍스트 입력 필드
- 평가 결과 CSV 다운로드

### 5.3 Non-goals (명시적 제외)
- **로그인/회원가입 시스템** - 평가장에서 바로 접속하므로 불필요
- **모바일 앱** - PC/노트북 웹 전용
- **복잡한 권한 관리** - 단순 구조 유지
- **알림/푸시 기능** - 평가장에서 동시 사용하므로 불필요

## 7. 해결 가설과 실험

### 7.1 가설

> 만약 **평가위원이 웹에서 직접 점수를 입력하면**, **집계 오류가 0%에 가까워질 것이다**. 왜냐하면 자동 계산이 수작업 계산의 실수를 제거하기 때문이다.

### 7.2 실험 루프

```
실험 → 관측 → 학습 → 다음 가설
```

| 단계 | 내용 |
|------|------|
| 실험 | 5명의 평가위원으로 모의 평가 진행 |
| 관측 | 집계 정확도, 출력 성공률, 사용 편의성 피드백 |
| 학습 | 오류 발생 시 원인 분석, UI 개선점 도출 |
| 다음 | 개선된 버전으로 실제 평가 적용 |
